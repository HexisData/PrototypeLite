Prototype Lite Database : hq_Recon Schema
hq_Recon Schema
Description

A simple data reconciliation engine. To use:

  • Populate t_Map with database objects to be reconciled and associated column mappings.
  • Execute usp_Execute against the reconciliation targets.
  • Read any reconciliation breaks from v_Break.
Tables
TableDescription
hq_Recon.t_BreakThe table that receives reconciliation breaks.
hq_Recon.t_MapA mapping table relating reconciliation columns and indicating whether a given column should be reconciled.
hq_Recon.t_TestLeftA table of typed test data for demonstration of the Recon module.
hq_Recon.t_TestRightA table of text test data for demonstraction of the Recon module.
Views
ViewDescription
hq_Recon.v_BreakA view over t_Break providing extra contextual information. Note that this view contains only reconciliation breaks.
Stored Procedures
Stored ProcedureDescription
hq_Recon.usp_ExecuteExecutes reconciliation against a designated pair of tables or views.
hq_Recon.usp_TestGenerateGenerates a rowset of test data for the Recon module.
hq_Recon.usp_TestResetResets the Recon module with fresh test data.
